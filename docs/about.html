The Loop

OmniForm is a single-page web app that turns any device with a camera into a real-time biomechanics coach. There is no text input anywhere — it's just a camera and one glowing button.

  1. Stand in frame. A live cyan skeleton locks onto your body via on-device MediaPipe BlazePose. Knee angles + peak angular velocity stream live to the HUD.
  2. Hold the capture button and perform any movement — a soccer strike, a squat, a walking pattern, a sit-to-stand. The app records 4 seconds of video + audio.
  3. Release. A 4-stage progress indicator (Pose → Physics → Coach → Voice) walks you through the real backend pipeline as Gemini 3.5 Flash analyzes the clip and Veo 3.1 generates a correction reference clip.
  4. ~15 seconds later your captured video plays back with server-computed overlay markers (warning rings, force vectors, joint labels) tracking your body via MediaPipe. The Chirp 3 HD voice delivers a two-sentence coaching critique. Then a "Correct-Form Reference" tile slides in with the Veo-generated clip animating you in your setting executing the corrected technique.

Architecture

Hero + data-flow visuals above generated by Google Imagen 4 Ultra via ai.models.generateImages() on the same @google/genai SDK powering the live app. Every visual asset in this project is Google-AI-generated.

Two services, both on Cloud Run. The user's recorded video never persists on the server — it's replayed from the browser's in-memory blob.

Google AI Stack

Multimodal video analysisGemini 3.5 Flash via @google/genai Interactions API — typed steps, multimodal video input, server-side history
Agent orchestrationGoogle Agent Development Kit (ADK) — LlmAgent with Zod-validated structured output, Runner.runEphemeral, InMemorySessionService
Reference-video generationVeo 3.1 Fast (veo-3.1-fast-generate-preview) image-to-video — takes the user's first video frame as the reference so the output animates the same person in the same setting
Voiced coachingGoogle Cloud Text-to-Speech · Chirp 3 HD (en-US-Chirp3-HD-Charon) with browser SpeechSynthesis fallback
On-device pose detectionMediaPipe BlazePose (@mediapipe/tasks-vision) — GPU-accelerated WASM, never leaves the device
RuntimeGoogle Cloud Run (frontend + backend) — serverless, autoscale, graceful SIGTERM drain, non-root containers
ObservabilityCloud Logging via JSON stdout with severity field
SecretsSecret Manager bound via --update-secrets (never --set-env-vars)
Data flywheelBigQuery-shape JSONL — every analysis writes a row ready for bq load

Activity-Agnostic

The Gemini prompts are written to handle any human movement, not just sport skills:

Sport skills

Soccer strike, jump shot, golf swing, deadlift, sprint, climbing, swim stroke.

Gait

Walking, running, stair climb, sit-to-stand, post-injury return-to-walk.

Posture

Sitting form, standing alignment, lifting mechanics, overhead carry.

Rehab / PT

Range-of-motion work, asymmetric load detection, knee-valgus tracking.

Try It

OmniForm needs a Gemini API key (free tier works) and Node 20. The full live app — Cloud Run + Veo + Chirp 3 HD — deploys in one command.

Run Locally (2 minutes)

git clone https://github.com/vnmoorthy/Omniform.git
cd Omniform

# 1. backend
cd backend
cp .env.example .env        # add GEMINI_API_KEY (and optionally GOOGLE_TTS_API_KEY)
npm install
node --env-file=.env index.js   # http://localhost:8080

# 2. frontend (separate terminal)
cd frontend
cp .env.local.example .env.local
npm install
npm run dev                  # http://localhost:3000

Deploy to Google Cloud Run (5 minutes)

# Pre-reqs: gcloud auth login + billing-enabled GCP project +
# backend/.env populated with GEMINI_API_KEY (and optionally GOOGLE_TTS_API_KEY).

PROJECT_ID=your-project-id ./deploy.sh

The script enables APIs, creates Secret Manager entries (omniform-gemini, omniform-tts, omniform-demo-token), deploys both services to Cloud Run, locks the backend's CORS to the frontend URL, and prints the final URLs.

Note for judges: the live Cloud Run deploy needs a Google Cloud project with the Gemini API, Veo, and Cloud Text-to-Speech APIs enabled. The repo includes deploy.sh for a one-command production deploy. To preview the UI without deploying, run the local steps above.

Privacy + Security

  • Recorded video never persists on the server. The result screen plays back the in-memory blob via URL.createObjectURL.
  • Veo correction clips + TTS audio are held for at most 1 hour, then swept.
  • Metrics are sanitized at the API boundary — only finite numbers in known ranges pass through, closing the prompt-injection class.
  • Rate-limited (10 analyses/min/IP, 3 Veo kickoffs/min/IP) and gated by an optional X-Demo-Token shared secret.
  • Non-root Cloud Run containers, graceful SIGTERM drain, 0.0.0.0 bind.
  • Full audit in SECURITY.md.
