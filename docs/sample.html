In the live app, Veo 3.1 Fast takes the first frame of your captured clip as the reference image and animates an ~8-second slow-motion video of you, in your setting, executing the corrected technique. The image above stands in for that video on this static page.

What's happening behind this screen
  1. Frontend uploads the 4-second clip + measured MediaPipe metrics to ai.models.generateContent on Gemini 3.5 Flash with a Zod-validated responseSchema.
  2. An ADK Director Agent consumes the multimodal output and emits the four-field JSON above: agentAudioScript, omniVideoPrompt, overlayPlan, correctionVideoPrompt.
  3. The script is voiced by Cloud Text-to-Speech · Chirp 3 HD (en-US-Chirp3-HD-Charon).
  4. The captured clip's first frame is sent to Veo 3.1 Fast via ai.models.generateVideos as the I2V reference — produces an 8-second mp4 of the same athlete performing the correction.
  5. If Veo is unavailable, Imagen 4 Ultra generates a still image of the corrected form as the fallback (also via @google/genai).
